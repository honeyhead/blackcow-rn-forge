# Mobile Team Orchestration

React Native 앱을 여러 에이전트로 나눠 작업할 때 쓰는 스킬입니다.

## 기본 원칙

1. `ios/`, `android/`, `src/` 수정 범위를 분리해요.
2. 네비게이션, 타입, API 계약은 먼저 합의해요.
3. 네이티브 설정 파일은 한 명이 소유하도록 해요.
4. QA와 release는 마지막이 아니라 중간부터 병렬로 붙여요.

## 추천 역할

- native-core
- feature-dev
- ui-dev
- backend-dev
- qa-release
