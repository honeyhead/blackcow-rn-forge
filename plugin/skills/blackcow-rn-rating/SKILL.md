# Mobile Rating Operations

React Native 앱의 in-app review timing, review monitoring, rating recovery 운영을 다루는 스킬입니다.

## 핵심 원칙

1. review prompt는 빈도가 아니라 만족 순간 정확도가 중요해요.
2. 부정 피드백을 숨기지 말고 store review와 support 경로를 분리해야 해요.
3. 평점은 growth 지표이면서 support/ops 지표예요.
4. locale과 release version별 평판 차이를 같이 봐야 해요.

- negative feedback routing
- review monitoring
