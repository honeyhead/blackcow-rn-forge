# Mobile Incident Response

React Native 앱의 severity 판단, rollback, incident comms, postmortem을 다루는 스킬입니다.

## 핵심 원칙

1. 알람과 incident는 같은 개념이 아니므로 severity 기준을 먼저 정해요.
2. rollback 판단은 기술 지표와 결제/인증 영향도를 함께 봐야 해요.
3. hotfix는 incident를 끝내는 수단이지 postmortem을 대신하지 않아요.
4. support, release, observability 문서는 incident 체계에서 연결돼야 해요.

## 다루는 영역

- severity matrix
- rollback criteria
- incident command
- user comms
- postmortem follow-up
