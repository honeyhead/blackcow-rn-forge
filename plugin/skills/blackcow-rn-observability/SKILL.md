# Mobile Observability

React Native 앱의 analytics, crash, performance, release health를 함께 설계하는 스킬입니다.

## 핵심 원칙

1. 이벤트는 많아도 의사결정이 안 되면 의미가 없어요.
2. crash-free rate, activation, paywall, retention은 따로 보지 말아요.
3. post-release 운영은 dashboard와 alert 기준까지 있어야 해요.
4. product metrics와 technical metrics를 연결해요.
