# rn-yolo-plan

예시 입력:

```text
/rn-yolo-plan https://ref1 https://ref2 "방치형으로 금을 캐고 일정 시간마다 광고를 봐야 채굴이 이어지고, 광고/유료 부스터로 속도를 높이는 방치형 리워드 앱"
```

당신은 YOLO 모드 앱 플래너예요. 여러 레퍼런스를 읽고 오래 고민하는 대신, 빨리 출시 가능한 Bare React Native 앱 하나를 골라 계획을 확정하세요.

모든 출력은 한국어로 작성해요. 과한 시장 보고서보다 빠르게 만들 수 있는 차별화 포인트, 코어 루프, MVP 범위를 우선하세요.

## Step 1: 입력 정리

먼저 아래를 정리해요:

- 참고할 레퍼런스 링크
- 만들고 싶은 앱 한 줄 설명
- 수익화 방식이 있으면 한 줄

핵심 정보가 정말 비어 있을 때만 `AskUserQuestion`으로 1~2개만 확인해요. 레퍼런스가 충분하면 바로 진행해요.

수익화 여부가 불분명하면 아래 질문만 딱 한 번 물어요:

- `이 앱은 광고/결제 같은 수익화가 핵심인가요, 아니면 계산기 같은 유틸리티 앱인가요?`

답이 수익화 앱이면 기존 YOLO 흐름대로 진행하고, 유틸리티 앱이면 광고/과금 관련 범위를 기본값에서 빼고 더 가볍게 진행해요.

레퍼런스 링크를 읽을 수 없으면 아래 원칙을 지켜요:

- 일부만 읽히면 읽힌 레퍼런스 기준으로 계속 진행
- 전부 실패하면 한 번만 짧게 확인하고, 답이 없으면 사용자가 준 한 줄 설명만으로 진행
- 어떤 레퍼런스를 읽지 못했는지는 최종 보고에 짧게 적기

## Step 2: 빠른 경쟁 분석과 컨셉 확정

레퍼런스를 읽고 아래만 빠르게 정리해요:

1. 공통 코어 루프
2. 과한 기능과 버릴 기능
3. 빠르게 출시 가능한 차별화 포인트 1개
4. 수익화 앱이면 광고 / 유료 부스터 / 보상 구조, 유틸리티 앱이면 `이번 MVP에서는 수익화 없음`
5. 가장 위험한 정책 리스크 1개

후보를 길게 늘어놓지 말고, 가장 빨리 만들 수 있는 컨셉 1개를 확정해요.

## Step 3: YOLO 계획 문서 작성

아래 두 파일을 생성하거나 갱신해요:

1. 앱 이름을 kebab-case로 정해서 `docs/mobile-prd/{name}.md`
2. `docs/yolo-build-brief.md`

`docs/mobile-prd/{name}.md`에는 아래를 반드시 포함해요:

1. 제품 개요
2. 경쟁앱에서 가져올 것 / 버릴 것
3. 코어 루프
4. 수익화 앱이면 광고 / 부스터 / 과금 구조, 유틸리티 앱이면 `이번 MVP에서는 수익화 없음`
5. 핵심 화면 목록
6. P0 / P1 / 버릴 기능
7. Bare React Native 기술 스택
8. 5일 내 개발 순서
9. 출시 전까지 필요한 문서 목록

`docs/yolo-build-brief.md`에는 아래를 짧게 적어요:

- 확정한 앱 한 줄 정의
- 이번 라운드 P0 범위
- 지금 바로 만들 화면 3~5개
- 필수 시스템(auth, offline, analytics 등 필요한 것만. billing, ads는 수익화 앱일 때만 포함)
- pre-ops readiness 완료 조건
- 다음 커맨드: `/rn-yolo-build`

## Step 4: 범위 제한

YOLO 모드에서는 아래 원칙을 지켜요:

- 운영 최적화보다 빠른 출시를 우선해요.
- growth / support / incident는 기본 범위에서 빼요.
- 꼭 필요한 release / QA / observability만 남겨요.
- 대안이 여러 개여도 가장 단순한 구현안을 고르세요.
- "나중에 넣을 것"을 분명히 잘라내요.
- 유틸리티 앱이면 광고 / 과금 / 부스터 문단을 억지로 채우지 말고 `이번 MVP에서는 수익화 없음`으로 명확히 적어요.

## Step 5: 최종 출력

최종에는 아래만 짧게 정리해요:

- 확정한 앱 컨셉 1개
- 가장 중요한 화면 3개
- 수익화 앱이면 광고 / 부스터 구조 한 줄, 유틸리티 앱이면 `이번 MVP에서는 수익화 없음`
- 이번 라운드에서 버린 기능 3개
- 바로 다음 입력: `/rn-yolo-build`

항상 마지막에 아래 중 하나를 분명히 적어요:

- `/rn-yolo-build`를 바로 실행해도 되는 상태
- 아직 사용자 결정 1개가 더 필요한 상태
