# rn-ui-upgrade

모바일 앱 UI/UX 감사를 진행하고, 필요한 경우 직접 수정해요.

## 감사 범주

1. safe area 처리
2. 터치 타겟 크기
3. 로딩/빈 상태/에러 상태
4. 폼 입력 경험
5. 접근성 레이블
6. 스크롤 성능
7. 애니메이션 과부하
8. 다크 모드 일관성
9. 큰 글꼴/동적 텍스트 대응
10. 네비게이션 복귀 흐름

## 우선순위

- P0: 사용 불가, 탭 불가, 글자 안 보임
- P1: 전환 저하, CTA 발견 어려움, 로딩 혼란
- P2: 성능 저하, 레이아웃 불안정, 접근성 누락
- P3: 미세 UX 개선

## 수정 원칙

- RN 성능에 불리한 불필요한 래퍼를 줄여요
- 긴 리스트는 가상화 전략을 우선 검토해요
- `Pressable` 사용을 우선 검토해요
- inline style 남발보다 재사용 스타일을 우선해요
- 로딩 상태는 skeleton 또는 명확한 progress indicator를 제공해요

## 결과

수정 전에는 발견 수를 알려주고 범위를 정해요.
수정 후에는 변경 파일, 우선순위별 수정 수, 남은 수동 검증 항목을 정리해요.
